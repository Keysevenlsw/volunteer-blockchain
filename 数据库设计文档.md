# 数据库设计文档

## 1. 用户表 (users)

~~~sql
CREATE TABLE users (
    user_id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(100) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,   -- 密码加密后存储
    email VARCHAR(100) UNIQUE,
    avatar_path VARCHAR(255),              -- 头像图片存储路径
    organization_id INT,                   -- 每个用户只能加入一个公益组织
    role ENUM('volunteer', 'organization_admin', 'system_admin') NOT NULL,  -- 三种角色：志愿者、公益组织管理员、系统管理员
    join_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    total_points INT DEFAULT 0,            -- 用户积分总数
    FOREIGN KEY (organization_id) REFERENCES organizations(organization_id)
);
~~~

- **描述**：存储用户的基本信息，包括用户名、密码、头像、角色、积分等。
- **字段**：
  - `user_id`: 用户ID，主键。
  - `username`: 用户名。
  - `password_hash`: 加密后的密码。
  - `email`: 用户邮箱，唯一。
  - `avatar_path`: 用户头像路径。
  - `organization_id`: 用户所属公益组织ID。
  - `role`: 用户角色（志愿者、公益组织管理员、系统管理员）。
  - `join_date`: 加入日期。
  - `total_points`: 用户的积分总数。

## 2. 公益组织表 (organizations)

~~~sql
CREATE TABLE organizations (
    organization_id INT AUTO_INCREMENT PRIMARY KEY,
    organization_name VARCHAR(255) NOT NULL,
    organization_description TEXT,
    avatar_path VARCHAR(255)               -- 公益组织头像存储路径
);
~~~

- **描述**：存储公益组织的基本信息和头像。
- **字段**：
  - `organization_id`: 公益组织ID，主键。
  - `organization_name`: 公益组织名称。
  - `organization_description`: 公益组织描述。
  - `avatar_path`: 公益组织头像路径。

## 3. 积分记录表 (points_records)

~~~sql
CREATE TABLE points_records (
    record_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    organization_id INT,
    points INT,
    transaction_type ENUM('earned', 'spent'),  -- 增加或消耗积分
    source VARCHAR(255),  -- 积分来源，如服务活动
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id),
    FOREIGN KEY (organization_id) REFERENCES organizations(organization_id)
);
~~~

- **描述**：记录用户积分的变化，包括来源、类型和时间。
- **字段**：
  - `record_id`: 积分记录ID，主键。
  - `user_id`: 用户ID，外键。
  - `organization_id`: 公益组织ID，外键。
  - `points`: 积分数额。
  - `transaction_type`: 积分类型（增加或消耗）。
  - `source`: 积分来源（如活动）。
  - `created_at`: 记录创建时间。

## 4. 积分商城商品表 (products)

~~~sql
CREATE TABLE products (
    product_id INT AUTO_INCREMENT PRIMARY KEY,
    organization_id INT,
    product_name VARCHAR(255) NOT NULL,
    product_description TEXT,
    price INT NOT NULL,  -- 商品兑换所需的积分数量
    stock INT NOT NULL,  -- 商品库存
    image_path VARCHAR(255),  -- 商品图片的文件路径
    category_id INT,        -- 商品分类ID
    FOREIGN KEY (organization_id) REFERENCES organizations(organization_id),
    FOREIGN KEY (category_id) REFERENCES product_categories(category_id)
);
~~~

- **描述**：每个公益组织的积分商城商品信息。
- **字段**：
  - `product_id`: 商品ID，主键。
  - `organization_id`: 公益组织ID，外键。
  - `product_name`: 商品名称。
  - `product_description`: 商品描述。
  - `price`: 商品所需积分。
  - `stock`: 商品库存。
  - `image_path`: 商品图片路径。
  - `category_id`: 商品分类ID，外键。

## 6. 活动表 (activities)

~~~sql
CREATE TABLE activities (
    activity_id INT AUTO_INCREMENT PRIMARY KEY,
    organization_id INT,
    activity_name VARCHAR(255) NOT NULL,
    description TEXT,
    start_date DATETIME,           -- 服务起始日期
    end_date DATETIME,             -- 服务结束日期
    publish_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 发布日期
    location VARCHAR(255),
    contact_name VARCHAR(255),     -- 联系人姓名
    contact_phone VARCHAR(20),     -- 联系人电话
    category_tags VARCHAR(255),    -- 服务类别标签
    image_path VARCHAR(255),       -- 活动图片路径
    max_participants INT,          -- 最大参与人数
    current_participants INT DEFAULT 0,  -- 当前参与人数
    status ENUM('pending', 'completed', 'ongoing') DEFAULT 'pending', -- 活动状态
    FOREIGN KEY (organization_id) REFERENCES organizations(organization_id)
);
~~~

- **描述**：记录志愿服务活动的基本信息。
- **字段**：
  - `activity_id`: 活动ID，主键。
  - `organization_id`: 公益组织ID，外键。
  - `activity_name`: 活动名称。
  - `description`: 活动描述。
  - `start_date`: 活动开始日期。
  - `end_date`: 活动结束日期。
  - `publish_date`: 发布日期。
  - `location`: 活动地点。
  - `contact_name`: 联系人姓名。
  - `contact_phone`: 联系人电话。
  - `category_tags`: 活动类别标签。
  - `image_path`: 活动图片路径。
  - `max_participants`: 最大参与人数。
  - `current_participants`: 当前参与人数。
  - `status`: 活动状态（待审核、已完成、进行中）。

## 7. 活动参与表 (activity_participation)

~~~sql
CREATE TABLE activity_participation (
    participation_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    activity_id INT,
    earned_points INT,         -- 参与活动获得的积分
    participation_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id),
    FOREIGN KEY (activity_id) REFERENCES activities(activity_id)
);

-- 创建触发器，防止活动人数超过最大人数
DELIMITER $$
CREATE TRIGGER prevent_over_participation BEFORE INSERT ON activity_participation
FOR EACH ROW
BEGIN
    DECLARE max_participants INT;
    DECLARE current_participants INT;

    -- 获取当前活动的最大参与人数和当前参与人数
    SELECT max_participants, current_participants
    INTO max_participants, current_participants
    FROM activities
    WHERE activity_id = NEW.activity_id;

    -- 判断是否超出最大参与人数
    IF current_participants >= max_participants THEN
        SIGNAL SQLSTATE '45000' SET MESSAGE_TEXT = 'Activity participation limit exceeded';
    END IF;
END $$
DELIMITER ;
~~~

- **描述**：记录志愿者参与的活动和获得的积分。
- **字段**：
  - `participation_id`: 参与记录ID，主键。
  - `user_id`: 用户ID，外键。
  - `activity_id`: 活动ID，外键。
  - `earned_points`: 获得的积分。
  - `participation_date`: 参与日期。

## 8. 活动审核表 (activity_audits)

~~~sql
CREATE TABLE activity_audits (
    audit_id INT AUTO_INCREMENT PRIMARY KEY,
    activity_id INT,
    auditor_id INT,
    audit_status ENUM('pending', 'approved', 'rejected') NOT NULL DEFAULT 'pending', -- 正确的默认值设置
    audit_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (activity_id) REFERENCES activities(activity_id),
    FOREIGN KEY (auditor_id) REFERENCES users(user_id)
);
~~~

- **描述**：记录活动的审核状态。
- **字段**：
  - `audit_id`: 审核记录ID，主键。
  - `activity_id`: 活动ID，外键。
  - `auditor_id`: 审核员ID，外键。
  - `audit_status`: 审核状态（通过或拒绝）。
  - `audit_date`: 审核日期。

## 9. 支援活动完成情况表 (activity_completion)

~~~sql
CREATE TABLE activity_completion (
    completion_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    activity_id INT,
    completion_status ENUM('pending', 'approved', 'rejected'),
    completion_data TEXT,         -- 完成情况的详细描述
    submission_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(user_id),
    FOREIGN KEY (activity_id) REFERENCES activities(activity_id)
);
~~~

- **描述**：记录志愿者提交的活动完成情况及审核状态。
- **字段**：
  - `completion_id`: 完成情况ID，主键。
  - `user_id`: 用户ID，外键。
  - `activity_id`: 活动ID，外键。
  - `completion_status`: 完成状态（待审核、已通过、已拒绝）。
  - `completion_data`: 完成情况的详细描述。
  - `submission_date`: 提交日期。

## 10. JWT Tokens 表 (jwt_tokens)

~~~sql
CREATE TABLE jwt_tokens (
    token_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    token VARCHAR(1024) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    expires_at TIMESTAMP NOT NULL,
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);
~~~

- **描述**：存储用户的JWT tokens，支持后续身份验证。
- **字段**：
  - `token_id`: TokenID，主键。
  - `user_id`: 用户ID，外键。
  - `token`: JWT Token字符串。
  - `created_at`: 创建时间。
  - `expires_at`: 过期时间。

## 11. 用户修改审计表 (user_audit_logs)

~~~sql
CREATE TABLE user_audit_logs (
    log_id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT,
    action VARCHAR(255),  -- 操作内容，如修改用户名、联系方式等
    old_value TEXT,       -- 修改前的值
    new_value TEXT,       -- 修改后的值
    changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,  -- 修改时间
    modified_by INT,  -- 修改操作的人员ID（管理员）
    FOREIGN KEY (user_id) REFERENCES users(user_id)
);
~~~

- **描述**：记录用户信息修改的历史。
- **字段**：
  - `log_id`: 审计日志ID，主键。
  - `user_id`: 用户ID，外键。
  - `action`: 操作内容。
  - `old_value`: 修改前的值。
  - `new_value`: 修改后的值。
  - `changed_at`: 修改时间。
  - `modified_by`: 修改操作的人员ID。
